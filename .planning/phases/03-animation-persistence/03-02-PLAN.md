**1. Create `src/components/pet-svg/AnimatedPetSVG.tsx`:**

A `'use client'` component that wraps PetSVG with all animation behavior. This keeps animation state OUTSIDE PetSVG so React.memo on PetSVG still works (avoids Pitfall 6 from research).

```typescript
interface AnimatedPetSVGProps {
  petId: string;
  traits: PetTraits;
  size?: 'small' | 'medium' | 'large';
  className?: string;
}
```

Implementation:

a) **Hooks:** Call `usePageVisibility()` and `useReducedMotion()`.

b) **Unique animation offset (ANIM-07):** Use `seedrandom(petId)` to generate a deterministic negative delay for breathing: `-(rng() * 3.5)` seconds. This starts each pet at a different point in its breathing cycle. Compute this with `useMemo` keyed on `petId` so it doesn't recalculate on every render.

c) **Body-size-specific timing (ANIM-03):** Map `traits.bodySize` to breathing duration: small = 2.5s, medium = 3.5s, large = 4.5s. Compute with `useMemo` keyed on `traits.bodySize`.

d) **Blink scheduling (ANIM-02):** Use `useEffect` to schedule random blinks:
   - Only run when `isVisible && !reducedMotion`
   - Use `seedrandom(`${petId}-blink-${Date.now()}`)` for randomized (not deterministic) intervals
   - Schedule next blink after 3000 + rng() * 2000 ms (3-5 second range)
   - On blink trigger: set `isBlinking` state to true, then after 200ms set back to false
   - Clean up all timeouts on unmount or when visibility/motion preference changes
   - IMPORTANT: Use a ref to track timeout IDs so cleanup works correctly. Use a recursive scheduling pattern (schedule next blink after current blink completes) rather than setInterval.

e) **Determine animation state:** Compute `shouldAnimate = isVisible && !reducedMotion`.

f) **Render:** Return a `<div>` wrapper that:
   - Sets CSS custom properties as inline styles: `--pet-breathing-duration` and `--pet-breathing-delay`
   - Passes `className` to the wrapper div
   - Renders `<PetSVG>` inside with:
     - `traits={traits}`
     - `size={size}`
     - `className` built with string concatenation (no external cn utility needed):
       - Always include `'pet-svg-animated'`
       - Add `'pet-animate-breathing'` when `shouldAnimate` is true
       - Add `'pet-paused'` when `!isVisible` (tab hidden but not reduced motion -- pause instead of remove)
       - Add `'pet-blinking'` when `isBlinking` is true

g) **Pass `isBlinking` to ExpressionLayer:** The blink animation needs to target the eyes specifically. Add an `isBlinking` prop to ExpressionLayer (see below). Pass this through PetSVG.

WAIT -- PetSVG is memoized and we want to avoid breaking memo. Instead, apply the blink effect at the AnimatedPetSVG level using CSS. The `.pet-blinking` class on the SVG element combined with `.pet-blinking .pet-eyes` CSS selector will handle blinking by targeting elements with class `pet-eyes`. This approach means ExpressionLayer needs a `className="pet-eyes"` on its eye group wrapper.
